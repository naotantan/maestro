## T07: 性能テスト（応答速度）— 最優先

**目的**: 各APIエンドポイントが目標応答時間内に応答することを確認する

**受け入れ基準（仮）**:
- 通常リクエスト: p95 < 200ms
- DB参照系: p95 < 500ms
- 認証処理: p95 < 300ms

**使用ツール**: Python requests + 計測スクリプト or `ab`（Apache Benchmark）or `wrk`

**仕様書**: `2026-04-03-w9-performance-test.md`（次に作成）

---

## T08: 負荷テスト

**目的**: 同時接続時のシステム安定性確認（レート制限の動作含む）

**受け入れ基準（仮）**:
- 10並列リクエスト: 全て正常応答
- 100並列: レート制限（429）が正常動作
- エラー率 < 1%

**使用ツール**: `ab` or Python concurrent.futures

**仕様書**: T07と同一ファイルで対応

---

## T09: ロングランテスト

**目的**: 1時間以上の継続稼働でのメモリリーク・接続プール枯渇を確認

**受け入れ基準**:
- 1時間後もメモリ使用量が安定している
- DB接続が正常に維持されている

**仕様書**: T07と同一ファイルで対応（軽量版: 10分に短縮）

---

## T10: 機能テスト（結合）

2. XSSペイロードが適切にエスケープされた状態で保存・取得される

**仕様書**: `2026-04-03-w9-functional-test.md`
