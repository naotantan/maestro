**レート制限テスト実施方法（L03/L04）**:
開発環境では `RATE_LIMIT_MAX=1000` がデフォルトのため、認証専用エンドポイント（`/api/auth`）の
厳格なレート制限（15分あたり10回）を利用してレート制限動作を確認する。

```bash
# 認証エンドポイントへ連続11回リクエスト → 11回目以降が HTTP 429 になることを確認
for i in $(seq 1 12); do
  curl -s -o /dev/null -w "Request $i: %{http_code}\n" \
    -X POST http://localhost:3000/api/auth/login \
    -H "Content-Type: application/json" \
    -d '{"email":"test@example.com","password":"test"}'
done
```

または `RATE_LIMIT_MAX=20` を環境変数で設定してサーバーを再起動し、
GET /api/agents への連続21回リクエストで確認することも可能。

**T09 プロセス監視方法**:
```bash
# テスト開始前にAPIサーバーのPIDを記録
SERVER_PID=$(lsof -ti:3000)
echo "開始時 PID: $SERVER_PID"

# テスト終了後に同じPIDが存在するか確認
FINAL_PID=$(lsof -ti:3000)
if [ "$SERVER_PID" = "$FINAL_PID" ]; then
  echo "✅ プロセス安定稼働（PID変化なし: $SERVER_PID）"
else
  echo "❌ プロセス再起動またはクラッシュ検出（開始: $SERVER_PID → 終了: $FINAL_PID）"
fi
```

**メモリ使用量監視（参考値）**:

開始時と終了時で50%以上増加していないことを確認（参考値・絶対基準なし）。

## 測定方法

```python
import time
import statistics
import requests
import concurrent.futures

def measure_latency(url, headers, n=50):
    """n回リクエストしてp50/p95/p99を返す"""
    latencies = []
    for _ in range(n):
        start = time.time()
        resp = requests.get(url, headers=headers, timeout=10)
        elapsed = (time.time() - start) * 1000  # ms
        latencies.append(elapsed)
    latencies.sort()
    return {
        'p50': latencies[int(n * 0.50)],
        'p95': latencies[int(n * 0.95)],
        'p99': latencies[min(int(n * 0.99), n-1)],
        'min': min(latencies),
        'max': max(latencies),
    }
```
