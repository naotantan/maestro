# 要件定義書 — タスク直接実行API

**バージョン**: 1.0.0
**作成日**: 2026-04-04

---

## 1. 背景・目的

現状、エージェントへのタスク実行手段はハンドオフ経由のみ。
エージェントに直接プロンプトを投げ、結果を同期的に受け取れる `/api/tasks` を追加する。
これにより:
- ハンドオフの from_agent に実績（agent_task_sessions）が積まれる
- 単体でエージェントを呼び出せる汎用APIになる

---

## 2. 機能要件

| ID | 要件 | 優先度 |
|----|------|--------|
| FR-01 | エージェントIDとプロンプトを指定してタスクを実行できる | Must |
| FR-02 | 実行結果（output）を同期的に返す | Must |
| FR-03 | 実行記録を agent_task_sessions に保存する | Must |
| FR-04 | context を任意で渡せる | Should |
| FR-05 | テナント分離（company_id フィルタ）を維持する | Must |

---

## 3. 非機能要件

| ID | 要件 |
|----|------|
| NFR-01 | タイムアウトはエージェントの config.timeout（デフォルト120秒）に従う |
| NFR-02 | エージェントが無効（enabled=false）の場合は400を返す |
| NFR-03 | 既存APIへの破壊的変更なし |

---

## 4. 用語定義

| 用語 | 定義 |
|------|------|
| task | エージェントへの単発プロンプト実行 |
| output | エージェントが返したテキスト |
